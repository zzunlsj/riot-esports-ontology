# 서비스 시나리오 전체 정리

> 기술 아키텍처(_index.md)의 상위 레이어. 서비스 구조 · 유저 여정 · 기능 매핑을 정의한다.

---

## 1. 서비스 개요

### 한 줄 정의
> "경기 중 중요한 순간을 놓치지 않게 하고, 끝나면 왜 중요했는지 설명해 주는 LoL 관전 보조 시스템"

### 핵심 가치
- **Before 한타**: 자동 관전 유도 — "지금 봐야 해" 알림 + 최적 시점 포인트 제시
- **After 한타**: 유저 수준에 맞는 딥다이브 해설 — Casual부터 Analyst까지 같은 한타를 다르게 설명
- **Throughout**: 실시간 승부 흐름 시각화 — Win Probability 그래프로 게임의 흐름이 어디서 바뀌었는지 직관적으로 표시

## 2. 서비스 구조 (3-Phase)

### Phase 1 — Pre-game (경기 시작 전)

```
입력:
  - 양팀 챔피언 구성 (Draft 완료 시점)
  - 선수 최근 폼 지표 (KDA, 챔피언 숙련도, 최근 승률)
  - 팀 H2H 기록

처리:
  - Neptune: 챔피언 구성 카운터/시너지 관계 탐색
  - pgvectorscale: 유사 구성 대결 이력 K-NN 검색
  - LightGBM: Pre-game 승률 예측

출력 (클라이언트):
  - 팀A 예상 승률 (ex. 57%)
  - 구성 강점/약점 요약 ("이 구성은 초반 교전이 강하고 후반 한타가 약함")
  - 주목할 선수 매치업 (ex. "정글 1:1 대결이 이 경기의 핵심")
```

### Phase 2 — In-game (경기 진행 중)

```
입력 (1초 폴링):
  - 양팀 플레이어 위치, HP%, 마나%, 스킬 쿨타임
  - 골드·킬·타워·오브젝트 현황
  - 와드 설치/파괴 이벤트

처리 흐름:
  TimescaleDB 저장
    ├── LightGBM → Win Probability (10초 갱신)
    ├── LSTM + K-NN → 한타 발생 확률 P(fight) (연속 계산)
    └── Rule Engine → 공간 클러스터링 감지

한타 예측 발동 (P > 0.75):
  ① 관전 포인트 좌표 계산
     - 전투 중심(centroid) + 오브젝트 위치 가중치
     - information density 최대화 (key player 최대 가시 좌표)
  ② 클라이언트 알림 발송 (WebSocket)
     - 관전 카메라 좌표 + 이유 ("드래곤 앞 5v5 예상")
     - 예상 한타 유리팀 + 확률 ("팀A 우세 62%")

한타 진행 중:
  - Win Probability 실시간 변동 그래프
  - 한타 참전 인원 실시간 추적

한타 종료 감지:
  - teamfight_events 테이블 기록
  - 딥다이브 해설 생성 파이프라인 트리거
```

### Phase 3 — Post-fight (한타 종료 직후)

```
입력:
  - teamfight_events 스냅샷 (한타 전/중/후 상태)
  - pgvectorscale: 유사 과거 한타 top-3 검색

처리:
  - Bedrock: 유저 수준별 해설 생성 (Level 1~4)
  - Win Probability 스윙 계산 (한타 가치 수치화)

출력 (Level별):
  Level 1 | Casual
    → "A팀이 이겼어요. 드래곤을 먹고 3킬을 냈습니다. (+1,200골드 이득)"

  Level 2 | Regular
    → "레오나의 E-R 콤보로 징크스를 잡은 것이 터닝포인트.
        드래곤 버프로 팀A 골드 차이가 2,400 → 4,800으로 벌어졌습니다."

  Level 3 | Hardcore
    → "진입 타이밍 분석: 레오나가 플래시 없는 상태에서 진입.
        상대 징크스의 W 쿨타임(8초)을 확인하고 들어간 판단은 적절.
        하지만 나미의 R을 맞으면서 딜 교환 효율이 -15% 저하.
        대안: 오브젝트 라인에서 교전했다면 포지션 이득 가능."

  Level 4 | Analyst
    → "한타 직전 시야 장악도: 팀A 와드 3개(오브젝트 주변 커버리지 78%)
        팀B는 역방향 시야 0. 정보 비대칭 상태에서 진입.
        이 한타의 게임 가치: Win Prob 51% → 74% (+23%p)
        유사 사례: 2025 LCK Spring T1 vs GEN.G 3경기 23:14 시점과
        구성·맥락 유사 (pgvectorscale 코사인 유사도 0.87)"
```

## 3. 유저 여정 (User Journey)

### 타겟 유저 세그먼트

| 세그먼트 | 정의 | 주요 니즈 | 체류 패턴 |
|---------|------|----------|----------|
| Casual Viewer | LoL 룰은 알지만 깊이 모름 | "지금 보면 되나?" "누가 이기고 있어?" | 간헐적 시청 |
| Regular Fan | 챔피언·스킬은 알고 있음 | "왜 이겼지?" "저 콤보 다시 보고 싶다" | 정기 시청 |
| Hardcore Fan | 메타·포지셔닝 이해 | "저 타이밍 선택이 맞았나?" | 풀 경기 시청 |
| Analyst / Coach | 전략·수치 분석 능력 있음 | "비교 데이터, 효율 수치" | 리뷰 중심 |

### 경험 흐름

```
[경기 시작 전]
  선수 폼 확인 → 구성 분석 → Pre-game 승률 확인
          ↓
[경기 시작]
  Win Probability 그래프 시작 (초기값)
          ↓
[경기 중]
  ┌─────────────────────────────┐
  │  (평온한 라인전 구간)         │
  │  Win Prob 서서히 변동        │  ← 유저는 다른 것 보다가
  └────────────┬────────────────┘
               │ P(fight) 상승
               ▼
  🔔 "한타 예상 — 드래곤 앞 5v5"
  → 관전 포인트 알림
  → 카메라 좌표 제시
          ↓
  [한타 진행]
  Win Prob 급변 구간 — 실시간 추이
          ↓
  [한타 종료]
  → 유저 레벨에 맞는 해설 팝업
  → "다시 보기" 버튼 (해당 구간 타임스탬프)
          ↓
[경기 종료 후]
  → 이 경기의 key fight 목록
  → 각 한타별 Win Prob 스윙 랭킹 ("이 경기를 결정지은 한타 TOP 3")
  → 수준 상향 제안 ("더 깊게 분석하려면 Level 3로 업그레이드")
```

## 4. 기능 매핑 (Feature × 기술 × 유저)

| 기능 | 유저 레벨 | 기술 컴포넌트 | 우선순위 |
|------|----------|--------------|---------|
| Pre-game 승률 예측 | All | Neptune + pgvectorscale K-NN | P1 |
| 한타 예측 알림 | All | LSTM + Rule + K-NN 앙상블 | P1 |
| 관전 포인트 좌표 제시 | All | Information Density Algorithm | P1 |
| Win Probability 그래프 | All | LightGBM (10초 인터벌) | P1 |
| 한타 결과 요약 (Casual) | Level 1 | Bedrock + 결과 템플릿 | P1 |
| CC콤보·킬 분석 (Regular) | Level 2 | Bedrock + 이벤트 체인 분석 | P2 |
| 포지셔닝·타이밍 분석 | Level 3 | Bedrock + 포지션 벡터 비교 | P2 |
| 시야·수치·유사경기 비교 | Level 4 | Bedrock + pgvectorscale RAG | P3 |
| 한타 가치 스윙 수치 | Level 2+ | Win Prob 전후 차분 | P2 |
| 유사 프로 경기 사례 | Level 4 | pgvectorscale 코사인 유사도 | P3 |
| 경기 결정 한타 TOP 3 | All | teamfight_value 정렬 | P2 |
| 챔피언 구성 상성 설명 | All | Neptune 그래프 탐색 | P2 |

```
Phase 0 ✅ 완료 (2026-04-29 Gate A 통과)
  - 온톨로지 구조 설계 ✅
  - TimescaleDB + pgvectorscale 프로토타입 ✅
  - Match-v5 백필 2,000경기 / team_state 106,566행 ✅
  - Win Probability EXP004 LightGBM AUC 0.8353 ✅
  - Teamfight Outcome EXP-TF-001 XGBoost Acc 83.4% / AUC 0.91 ✅
  - Composition Embedding 1,572 nodes / PCA 64dim ✅

Phase 1 (MVP): 핵심 기능  ← 현재 (Gate B 진행 중)
  - 한타 감지 rule-based + model_signal 로직 ✅ (완료)
  - Win Probability LightGBM (과거 데이터 학습) ✅ (완료)

  - Live Stats WebSocket 실 연결 ⚫ Gate B P0
  - Pre-game 승률 예측 (pgvectorscale K-NN) ⚫ Gate B (EXP-PG-001)
  - 웹 오버레이 MVP ⚫ 미착수

Phase 2: 예측 고도화
  - LSTM 한타 예측 모델
```
